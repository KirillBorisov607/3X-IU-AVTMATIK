# 3x-ui Автоматическая Установка

Единый скрипт установки для панели [3x-ui](https://github.com/mhsanaei/3x-ui) и агрегатора подписок.

Один скрипт делает всё: выбор языка, выбор режима, защита сервера, установка панели спрятанной за nginx, управление подписками.

## Требования

- Ubuntu 20.04 / 22.04 / 24.04 или Debian 11 / 12
- Доступ root

## Запуск

На любом сервере:

```bash
bash <(curl -Ls https://raw.githubusercontent.com/KirillBorisov607/3X-IU-AVTMATIK/master/install.sh)
```

Скрипт спросит:

1. **Язык** — Русский или English
2. **Режим** — Панель или Агрегатор
3. Если Агрегатор — Установить или Добавить подписку

## Режим: Панель

Устанавливает 3x-ui с полной защитой сервера. Панель прячется за nginx и доступна только через секретный рандомный путь по HTTPS.

Что настраивает скрипт:

| Компонент | Описание |
|-----------|----------|
| sysctl | SYN cookies, защита от спуфинга, логирование мартиан, тюнинг производительности |
| SSH | Нестандартный порт, только ключи, MaxAuthTries 3 |
| UFW | Запретить всё, разрешить только нужные порты, rate limit на SSH |
| iptables | Блок NULL/XMAS/FIN сканов, защита от SYN flood, дроп невалидных пакетов |
| Fail2ban | SSH (3 попытки, бан 24ч), панель (5 попыток) |
| nginx | HTTPS реверс-прокси на отдельном порту, возвращает 444 на все пути кроме секретного |
| 3x-ui | Официальный установщик, привязан только к localhost, секретный base path |
| Автообновления | Автоматические security-патчи |

Доступ к панели после установки:

```
https://IP_СЕРВЕРА:ПОРТ_NGINX/СЕКРЕТНЫЙ_ПУТЬ/
```

Секретный путь генерируется случайно при установке. Прямой доступ к внутреннему порту панели заблокирован — только nginx может до него достучаться.

После установки:
1. Проверить SSH на новом порту не закрывая текущую сессию
2. Удалить старый SSH порт: `ufw delete allow 22/tcp`
3. Удалить файл с данными: `rm /root/3xui-credentials.log`
4. Добавить клиентов в панели, скопировать их Subscription URL
5. Добавить Subscription URL в агрегатор

## Режим: Агрегатор

### Вариант 1 — Установить агрегатор

Разворачивает сервис агрегатора подписок. Он собирает подписки со всех настроенных 3x-ui серверов и объединяет их в одну ссылку.

После установки используй "Добавить подписку" для добавления серверов по одному.

### Вариант 2 — Добавить подписку

Добавляет новый сервер 3x-ui в существующий агрегатор. Запрашивает название сервера, URL подписки и префикс для прокси. Перезапускает сервис автоматически.

Итоговая ссылка подписки:

```
http://IP_АГРЕГАТОРА:ПОРТ/sub/ТОКЕН
```

Добавь эту ссылку в любой прокси-клиент (v2rayN, Hiddify, Sing-box и др.) — он вернёт прокси со всех серверов.

## Как получить Subscription URL из 3x-ui

1. Открыть панель
2. Зайти в настройки клиента
3. Скопировать ссылку из вкладки Subscription

Выглядит так: `https://IP:PORT/PATH/sub/CLIENT-UUID`

## Заметки по безопасности

- Порт панели привязан к localhost — снаружи недоступен
- nginx стоит перед ней и проксирует только секретный путь
- Все остальные пути возвращают 444 (TCP close, без ответа)
- Fail2ban следит за логами nginx на предмет неудачных входов
- Токен агрегатора сравнивается в constant-time для защиты от timing attacks
- Агрегатор возвращает 404 при неверном токене (не раскрывает существование эндпоинта)
- Генерация токена: `openssl rand -hex 32`
