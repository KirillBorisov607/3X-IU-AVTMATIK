# 3x-ui Автоматическая Установка

Скрипт автоматической установки и защиты сервера для панели [3x-ui](https://github.com/mhsanaei/3x-ui), а также агрегатор подписок — объединяет несколько серверов в одну ссылку.

## Требования

- Ubuntu 20.04 / 22.04 / 24.04 или Debian 11 / 12
- Доступ root
- Python 3.10+ (для агрегатора)

## Быстрый старт

### 1. Установить 3x-ui на каждый сервер

Загрузить скрипт и запустить на каждом VPS:

```bash
scp install.sh root@IP_СЕРВЕРА:/root/
ssh root@IP_СЕРВЕРА
bash install.sh
```

Скрипт спросит:
- Новый SSH порт (по умолчанию: 2222)
- Порт панели (по умолчанию: 2053)
- Имя пользователя и пароль панели

Всё остальное настраивается автоматически.

### 2. Развернуть агрегатор подписок

Агрегатор запускается на одном сервере и объединяет подписки со всех 3x-ui серверов в одну ссылку.

**Сначала отредактировать конфиг:**

```bash
nano aggregator/config.yaml
```

Задать токен и добавить URL подписки каждого сервера:

```yaml
token: "длинный-случайный-токен"   # openssl rand -hex 32

servers:
  - name: "Сервер-1 (US)"
    sub_url: "http://1.2.3.4:2053/путь-панели/sub/uuid-клиента"
    prefix: "US"
    verify_ssl: false

  - name: "Сервер-2 (DE)"
    sub_url: "http://5.6.7.8:2053/путь-панели/sub/uuid-клиента"
    prefix: "DE"
    verify_ssl: false
```

**Где найти ссылку подписки в 3x-ui:**
1. Открыть панель
2. Зайти в настройки клиента
3. Скопировать subscription link из вкладки "Subscription"

**Развернуть агрегатор:**

```bash
scp -r aggregator/ root@IP_АГРЕГАТОРА:/root/aggregator/
ssh root@IP_АГРЕГАТОРА
cd /root/aggregator
bash setup.sh
```

### 3. Добавить одну ссылку в клиент

```
http://IP_АГРЕГАТОРА:8080/sub/ВАШ_ТОКЕН
```

Добавить эту ссылку в любой прокси-клиент (v2rayN, Hiddify, Sing-box и др.). Он вернёт прокси со всех серверов сразу.

## Что делает install.sh

| Шаг | Описание |
|-----|----------|
| Обновление системы | apt upgrade, установка базовых утилит |
| sysctl | SYN cookies, защита от спуфинга, отключение ICMP редиректов |
| SSH | Нестандартный порт, только ключи, MaxAuthTries 3 |
| Файрволл UFW | Запретить всё, разрешить только нужные порты, rate limit |
| iptables | Блокировка NULL/XMAS/FIN сканов, защита от SYN flood |
| Fail2ban | SSH (бан 24ч после 3 попыток), панель (бан после 5 попыток) |
| Установка 3x-ui | Официальный установщик + настройка через CLI |
| Автообновления | Автоматические security-патчи |

После установки учётные данные сохраняются в `/root/3xui-credentials.log`. Удалить этот файл после переноса данных в менеджер паролей.

## Что делает агрегатор

- Загружает содержимое подписки с каждого настроенного 3x-ui сервера
- Декодирует base64, объединяет списки прокси, кодирует обратно в base64
- Кэширует результат на 5 минут (настраивается), чтобы не нагружать серверы
- Отдаёт результат по адресу `/sub/<токен>`
- Ограничивает количество запросов с одного IP
- Возвращает 404 при неверном токене (не раскрывает существование эндпоинта)

## Структура файлов

```
.
├── install.sh                     # Скрипт установки сервера
└── aggregator/
    ├── app.py                     # Flask приложение агрегатора
    ├── config.yaml                # Список серверов и токен
    ├── requirements.txt           # Python зависимости
    ├── setup.sh                   # Скрипт развёртывания агрегатора
    └── sub-aggregator.service     # systemd unit файл
```

## Заметки по безопасности

- Генерировать сильный токен: `openssl rand -hex 32`
- Агрегатор возвращает 404 (не 403) при неверном токене
- systemd сервис работает под выделенным непривилегированным пользователем
- `verify_ssl: false` нужен если панель использует самоподписанный сертификат

## Чеклист после установки

1. Проверить SSH на новом порту, не закрывая текущую сессию
2. Удалить старый SSH порт из UFW: `ufw delete allow 22/tcp`
3. Удалить файл с учётными данными: `rm /root/3xui-credentials.log`
4. Добавить клиентов в панели 3x-ui и скопировать их ссылки подписки
5. Добавить ссылки в `aggregator/config.yaml`
